## 2023-11-20 - Unnecessary `set:html` for dynamic styling
**Vulnerability:** The `HLine.astro` component used `set:html` with string interpolation to render the `tagline` prop and apply dynamic classes based on `center` prop (`const text = \`<span class="px-5 pr-6 ${center == 'c' ? 'shrink-0' : ''}">${tagline}</span>\`; <span set:html={text} />`). This is a dangerous pattern that bypasses default escaping and can lead to Cross-Site Scripting (XSS) if the `tagline` prop contains unescaped user input.
**Learning:** The root cause is a misunderstanding of how to conditionally apply classes or structure in Astro JSX. Using string building and `set:html` is entirely unnecessary here.
**Prevention:** Use standard Astro JSX for structure and dynamic classes directly (e.g., `<span class={\`px-5 pr-6 ${center == 'c' ? 'shrink-0' : ''}\`}>{tagline}</span>`). This allows the framework to securely escape the content of `tagline` by default while still achieving dynamic styling.
